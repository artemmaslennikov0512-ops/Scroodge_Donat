# ADR 0001: Платёжный шлюз (Stripe)

**Статус:** Принято  
**Дата:** 2026-03

## Решение

Использовать абстракцию `PaymentGateway` (порт) с единственной реализацией **Stripe**. Paygine и другие провайдеры в проекте не используются.

## Контекст

- Сайт донатов принимает платежи только через Stripe (Payment Intents, Elements).
- Логика приёма донатов не должна зависеть от конкретного провайдера в коде use case; зависимость инвертирована через порт.

## Варианты

1. Прямые вызовы Stripe в API routes — жёсткая связь.
2. Порт PaymentGateway + адаптер Stripe — выбран.
3. Event-driven — избыточно для текущего объёма.

## Последствия

- Соблюдение Open/Closed для платёжного модуля.
- Тесты: mock PaymentGateway в use case.
- Webhook: проверка подписи в адаптере; в production без секрета — отказ.
